# Shared browser-demo lib

Code and assets shared by the three published in-browser surfaces — `/playground/`
(edit Ruby → see emitted code, any target), `/studio/` (edit Ruby → run the
emitted TypeScript blog live), and `/ide/` (the analyzer as a browser IDE —
inferred-type hover, completion, and the coverage ledger over a whole Rails app).
Factored out in rung D, Phase 4 of
`docs/browser-demo-plan.md` ("share code, separate surface").

| File | Role | Tracked |
|---|---|---|
| `transpile.mjs` | C-ABI compiler driver (`loadCompiler`) + browser helpers `loadDefaultCompiler` / `loadFixture` (resolve the assets below via `import.meta.url`) | yes |
| `wasi-shim.mjs` | minimal `wasi_snapshot_preview1` shim — web-platform APIs only, runs unchanged in Node and the browser | yes |
| `editor.js` | editor abstraction: Monaco via CDN, `<textarea>`/`<pre>` fallback (`createEditor`, `createOutputView`) | yes |
| `tree.js` | generic collapsible file-tree widget (`buildTree`, `allDirPaths`, `renderTree`) | yes |
| `bundle.mjs` | in-browser TS→JS bundler (`loadBundler`): esbuild-wasm (CDN-loaded) + virtual-FS/CDN-external plugin; `bundle()` for the app, `bundleTests()` for the per-file test bundles; used by `/studio/` | yes |
| `test-runtime.mjs` | in-browser test harness (rung D.2): `node:test`/`node:assert` shims, in-memory `Db` over sqlite-wasm, `setupTestDb`, and per-spec entry sources — injected at bundle time so the emitted suite stays byte-identical to CI | yes |
| `sourcemap.mjs` | minimal source-map v3 consumer (`originalPositionFor`, `normPath`); maps a failing test back to its Ruby source line via the emitted `.test.ts.map` (rung D.2 Phase 9) | yes |
| `app-host.mjs` | runs the bundled app in an iframe via a service worker (`createAppHost`): registers `/studio/sw.js`, pushes the file map, `update()` (first mount / full reload) + `hotSwap()` (push bundles, post the running app a swap request → Turbo-morph, no reload) | yes |
| `roundhouse_wasm.wasm` | the compiler (~4.3 MB) — **built, not committed**: gitignored; CI's `build-wasm` job builds it from source (via `../vendor/ruby-rbs-sys/`), locally `../build.sh` | no (gitignored) |
| `fixture.json` | seed app (real-blog), generated by `../browser-spike/gen-fixture.mjs`; CI regenerates it from the freshly-built fixture | yes |
| `engine.mjs` | unified wasm engine driver: one instance + one generic `callExport(name, obj)` over the C-ABI, hosting every entry point — `transpile` plus the analyze query surface (`analyze_app`/`complete`/`type_at`/`related_files`/`traceroute`/`trace_targets`) — so a single worker backs both emit and analyze | yes |
| `worker.mjs` | shared compiler Web Worker: owns one `engine.mjs` instance off the UI thread; `{id, op, args}` RPC; backs both `/playground/` (transpile) and `/ide/` (analyze) so a multi-second pass never blocks the tab | yes |
| `wasm-client.mjs` | main-thread client for `worker.mjs`: the RPC plus a watchdog that terminates + respawns the worker on a hang/trap, so a survey-degraded app renders a diagnostic instead of freezing the tab | yes |
| `analyzer.mjs` | analysis driver wrapping the query C-ABI (`analyze_app`/`complete`/`type_at`/`related_files`/`traceroute`) as a clean object over the wasm's last-good snapshot; drives `/ide/` and Node validation | yes |
| `apps.json` | app-picker manifest (`default` + a `{name, label, src}` list) for the seed-app selector shared by the surfaces | no (gitignored, CI-generated) |
| `app-lobsters.json` | Lobsters benchmark app bundle (a seed app for the picker) | no (gitignored, CI-generated) |
| `app-mastodon.json` | Mastodon app bundle (a seed app for the picker) | no (gitignored, CI-generated) |

## Serving / paths

The surfaces import `../lib/…` and resolve the binary + fixture relative to this
module, so the **served root must be the parent** (`wasm/` locally, `_site/`
published) — not an individual surface dir. That mirrors how Pages serves the
tree: `_site/lib/`, `_site/playground/`, `_site/studio/` side by side.

```sh
python3 -m http.server 8099   # run from wasm/
open http://localhost:8099/playground/
open http://localhost:8099/studio/
```

No npm, no bundler, no COOP/COEP — every module is hand-authored ESM and the
only third-party code (Monaco) loads from a CDN with a local fallback.
